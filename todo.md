## 1. 函数类型语法统一（已完成）

**旧状态**：函数类型标注写 `(i32) -> i32`，而函数声明和闭包表达式写 `fun...`。

```nc
# 三者不一致
fun add(x: i32): i32 { ... }        # 声明
let f = fun(x: i32): i32 { x + 1 }  # 闭包表达式
let g: (i32) -> i32 = ...           # 函数类型标注 ← 不一致
```

**结果**：已改为 `fun(i32) i32`（Go 风格，无冒号），使三者统一为 `fun` 前缀。

| 角色 | 旧语法 | 新语法 |
|------|--------|--------|
| 函数类型 | `(i32) -> i32` | `fun(i32) i32` |
| | `() -> str` | `fun() str` |
| | `(i32, str) -> bool` | `fun(i32, str) bool` |

**实现边界**：
- `parser.py` — `_parse_type` 中函数类型分支检测 `FUN` 关键字
- 内部类型字符串仍用 `fn(i32)->i32`
- 旧 `(i32) -> i32` 不向前兼容
