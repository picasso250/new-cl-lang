# NC 编译器 — 原则

> 每至多 10 个 BDD case 通过后，暂停，回头审视，自省不落 md。
> 必问：我们距离现有的 Go、Zig、Rust，按重要性排序还差在哪里？不要按特性数量排，按未来返工痛苦和语言成败风险排。
> 本文件只记录原则，不记录具体问题清单；具体问题由 case / worklog / issue 推动。

- **递归下降，一运算符类一层**。优先级从低到高：赋值 < 逻辑 < 比较 < 位 < 加减 < 乘除 < 一元 < 基本。加运算符时找准位置插层。超过 10 层再议 Pratt。
- **case 驱动架构**。每个 case 逼出一个编译器能力，不提前设计未用到的能力。
- **先通后优**。链路未通之前不优化，链路一旦通，即刻自省。
- **多 pass，各司其职，不跨界**。每个 pass 只做一件事（建表、检查、出码）。信息收集与代码产出分离。可有多 pass，不可有一 pass 做两事。
